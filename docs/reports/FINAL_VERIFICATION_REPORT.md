# 글로벌 1만 명 AI 인프라 사업 타당성 전면 재검증 보고서

**작성일**: 2026-07-03  
**대상**: Vultr 기반 오픈소스 자체 구축 (Neon, vLLM, Modal, LiteLLM) vs 상용 API  
**결론 요약**: "자체 구축이 무조건 저렴하다"는 이전의 주장은 DB 원가만 고려한 오류였습니다. GPU 추론 원가를 포함한 전면 재계산 결과, **자체 구축은 고가(GPT-4급) 모델을 서빙할 때만 수익성이 있으며, 저/중가 모델 서빙 시에는 상용 종량제 API에 참패합니다.** 벤더 락인 리스크 역시 치명적입니다.

---

## 1. 워크로드 및 단가 정의 (실측/시장조사 기반)

막연한 추측을 배제하기 위해, 글로벌 1만 명 사용자의 워크로드를 시장 실측 데이터에 근거하여 모델링했습니다 [1] [2].

### 1.1. GPU 처리량 (vLLM Continuous Batching)
- **기준 GPU**: NVIDIA L40S 48GB (Vultr 베어메탈 8장 구성 시 **$0.848/GPU/hr**) [3]
- **처리량 실측**: Llama 3.1 8B 모델, FP8 양자화, Batch 32 기준 **실효 1,400 tokens/sec** (aggregate) [1] [2]

### 1.2. 1만 명 워크로드 가정
- **1인당 호출량**: 300콜/일 (Medium 시나리오)
- **호출당 토큰**: 1,500 토큰 (입출력 합)
- **월 총 토큰**: 약 135 Billion tokens
- **피크 집중도**: 업무 8시간 집중, 피크는 평균의 3배 가정

### 1.3. 상용 API 단가 비교군
- **저가 (gpt-4o-mini급)**: $0.30 / 1M tokens
- **중가 (혼합)**: $0.60 / 1M tokens
- **고가 (GPT-4급)**: $3.00 / 1M tokens

## 2. 전면 TCO 시뮬레이션 결과

GPU 추론, 파인튜닝, 3리전 DB 복제, API 게이트웨이, 운영 인건비(SRE 3명), 개발 상각비까지 모두 포함한 월 총소유비용(TCO)입니다.

### 2.1. 자체 구축 vs 상용 API (월 135B 토큰 처리 시)

| 항목 | 월 비용 (USD) | 비고 |
|---|---|---|
| **자체 구축 (Vultr L40S 335장)** | **$147,514** | 야간 유휴(Scale-down) 가동비율 56.7% 반영 |
| 상용 API 저가 ($0.30/M) | $40,500 | 자체 구축이 3.6배 비쌈 |
| 상용 API 중가 ($0.60/M) | $81,000 | 자체 구축이 1.8배 비쌈 |
| **상용 API 고가 ($3.00/M)** | **$405,000** | **자체 구축이 2.7배 저렴** |

> **분석**: 자체 구축 시 Cost-per-token은 약 **$1.09/M tokens**입니다. GPU를 피크 대비로 상주시켜야 하므로, 필연적으로 발생하는 야간/유휴 시간의 낭비가 단가를 끌어올립니다. 따라서 $1.09/M 이하로 팔리는 저가 상용 API와 경쟁하는 것은 구조적으로 불가능합니다.

### 2.2. 손익분기점 (Breakeven Point)

![자체구축 vs 상용 API 손익분기 곡선](breakeven_curve.png)

위 그래프에서 보듯, 1인당 하루 호출량이 아무리 늘어나도 **저/중가 API 선(초록, 주황 점선)은 자체 구축 선(남색 실선)을 넘지 못합니다.** 오직 고가 API 선(빨간 점선)만이 50콜/일 이상에서 자체 구축보다 비싸집니다.

## 3. 치명적 리스크: Vultr 벤더 락인 (Vendor Lock-in)

이전 보고서에서 누락된 가장 큰 리스크입니다. 자체 구축 원가의 **약 80%가 Vultr GPU 대여료**입니다. 오픈소스로 소프트웨어 종속성을 탈피하더라도, 하드웨어 종속성은 그대로 남습니다.

**Vultr GPU 단가 인상 시 월 원가 변화 (Medium 기준):**
- 현재 단가: $147,514
- **+20% 인상 시**: $171,000 (상용 API 대비 경쟁력 추가 하락)
- **+50% 인상 시**: $206,000

단일 클라우드 제공자에게 의존할 경우, 가격 인상 통보에 방어할 수단이 없습니다. 이는 "Sovereign(주권)" 인프라의 취지에 정면으로 위배됩니다.

---

## 4. 최종 결론 및 사업 방향 수정 제안

이상의 물리적, 재무적 검증을 종합할 때, 기존의 "오픈소스 + Vultr = 무조건 이익" 가설은 폐기해야 합니다. 사업이 성립하기 위한 유일한 조건은 다음과 같습니다.

1. **프리미엄 모델 타겟팅**: Llama 70B 등 대형 오픈소스 모델을 튜닝하여, GPT-4급($3.00/M) 상용 API를 대체하는 시장을 노려야 합니다.
2. **초고밀도 멀티테넌시 제어평면 구축**: 자체 구축의 약점인 "유휴 시간"을 없애기 위해, 1만 명의 요청을 빈틈없이 스케줄링하여 GPU 활용률을 90% 이상으로 끌어올리는 제어평면(Modal/Bedrock의 핵심 기술) 개발에 사활을 걸어야 합니다.
3. **멀티 클라우드 아키텍처**: Vultr 단일 종속을 피하고, CoreWeave, Lambda Labs 등 타 저가 GPU 클라우드 간 워크로드를 동적으로 옮길 수 있는 라우팅 레이어(OpenRouter 역할)가 필수적입니다.
