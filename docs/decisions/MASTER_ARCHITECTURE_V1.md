# Sovereign AI 인프라 플랫폼: 통합 마스터 아키텍처 및 단계별 실행 기준 (v1.0)

## 1. 아키텍처 설계 사상 및 목표
본 아키텍처는 단순한 클라우드 리셀링이나 단일 DB 서비스가 아닙니다. **"고객의 AI 워크플로우(데이터 적재 → 임베딩 → 에이전트 연산 → 외부/내부 모델 추론) 전체를 하나의 폐쇄적 데이터 흐름으로 묶어, 고객이 이탈할 수 없는 생태계(Lock-in)를 구축하는 것"**이 최종 목표입니다. 

이를 위해 하드웨어 제어권(Vultr 베어메탈, SmartNIC, SR-IOV)을 직접 확보하여 AWS 대비 1/10의 비용 효율을 달성하고, 5명의 엔지니어로 글로벌 1만 명 동시 접속을 무중단 운영할 수 있는 3대 자동화 봇(의존성 매핑, 시뮬레이션 컴파일러, eBPF 자가 치유)을 내재화합니다.

## 2. 4대 제품 레이어 및 시너지 구조
고객은 단일 API 게이트웨이(Cloudflare → Envoy)로 진입하며, 뒤에서 4개 제품이 유기적으로 데이터를 교환합니다. 데이터는 절대 외부로 나가지 않습니다.

| 레이어 | 역할 | 핵심 기술 및 스택 | 자체 구현 IP 및 수정 사항 |
|---|---|---|---|
| **L1: 제어 평면 (Control Plane)** | 인증, 정산, 라우팅, 거버넌스 | Go (정산/API), Rust/C++ (Envoy 필터) | 예약 잠금(Pre-auth), Envoy 인라인 토큰 카운터, 지오 라우팅 방화벽 |
| **L2: 스토리지 (Neon Core)** | 불변 데이터, 1초 브랜칭 | Rust (Pageserver/Safekeeper), MinIO | QPS 기반 오토스플릿, Safekeeper 샤딩, ArcSwap 락프리 족보 |
| **L3: 격리 연산 (Modal 대안)** | AI 에이전트 격리 실행 | Rust (Firecracker 오케스트레이터), Go | Scale-to-Zero 프록시, 토큰 버킷 주입, 50ms 스냅샷 복원 |
| **L4: GPU 서빙 (vLLM/OpenRouter)** | 자체 OSS 모델 및 외부 API 중계 | Python (vLLM), Rust (라우터) | vLLM APC, PagedAttention v2, S-LoRA 포인터 매핑, 주단위 검수 워커 |

## 3. 핵심 방어 및 운영 거버넌스 체계
원본 설계에서 누락되었던 핵심 방어막과 운영 체계를 우리 기준으로 재정립합니다.

### 3.1. 비용 및 정합성 방어
- **이그레스(Egress) 방화벽**: Cloudflare D1/KV를 활용한 글로벌 엣지 잔액 캐싱으로 단순 조회 이그레스 비용 0원 달성. 지오 라우팅으로 대륙 간 GPU 트래픽 원천 차단.
- **금융 먹튀 방어**: 외부 모델(L4) 호출 전 L1 제어 평면에서 잔액을 예약 잠금(Pre-auth Lock) 처리.
- **DB 폭발 방어**: 초당 10만 건의 정산 조회를 Envoy 인라인 토큰 카운터(L4)와 토큰 버킷 주입(L3)으로 분산시켜 중앙 DB 부하 소멸.

### 3.2. 스토리지 주권 및 확장성
- **R2 아카이빙 2트랙**: 라이브 데이터는 Vultr 로컬 MinIO 풀에만 상주. R2는 명시적 아카이브(R2-1)와 B2B 고객 LoRA 자산(R2-2, 5분 Presigned URL)으로 철저히 분리.
- **요금제 분리**: 100GB 단위 Hard 할당(고정비)과 바이트 단위 실시간 서버리스(가변비) 동시 지원.

### 3.3. 5인 운영 자동화 체계 (Bots)
- **형상 의존성 매핑 봇**: 코드 수정 시 L1~L4 전체 영향도 자동 분석 및 경고.
- **격리형 시뮬레이션 컴파일러**: AI/개발자가 수정한 코드를 상용 배포 전 샌드박스에서 k6 부하 테스트로 자동 검증.
- **eBPF 자가 치유 봇 (Cilium)**: 50ms 내 장애 노드 감지 및 자동 교체.
- **주단위 자동 벤치마크 워커**: 고객이 등록한 OSS 모델의 성능을 주단위로 자동 평가하여 대시보드 공시.

## 4. 단계별 실행 기준 (Roadmap)
모든 구현은 이 마스터 문서의 기준을 따르며, 단계별 완료 조건을 충족해야 다음으로 넘어갑니다.

### Phase 1: 기반 정합성 및 L1/L2 연동 (현재 진행)
- **1-1. T1 이중과금 결함 수정**: `store.go`의 `idempReserve/idempCommit` 로직 완결성 검증 및 반영.
- **1-2. L-5 외부 호출 인가 스텁 선 배치**: 모든 외부 LLM 호출 전 보안 인가 스텁(`CheckExternalLLMAuth`) 의무 배치.
- **1-3. Scale-to-Zero TCP 프록시 개발**: L3 연산을 위한 커스텀 패킷 홀딩 프록시 설계 및 구현 (Rust/Go).
- **1-4. L2 Compute Hook Handler 개발**: Neon 스토리지 이동 시 제어 평면과 통신하는 웹훅 수신부 구현.

### Phase 2: 인프라 몸통(관측성/백업) 및 L3/L4 전개
- **2-1. 관측성 및 백업 스택**: Prometheus/Grafana, pg_dump 크론 자동화, Thanos 메트릭 수집 세팅.
- **2-2. L3 Firecracker 연동**: Vultr 일반 K8s 환경에서 gVisor 1차 적용 후, 베어메탈 확보 시 Firecracker 오케스트레이터로 전환.
- **2-3. L4 하이브리드 라우팅**: Envoy 기반 외부 API 중계 및 고객 OSS 모델(vLLM) 라우팅 구현.

### Phase 3: 글로벌 스케일아웃 및 자동화 봇 가동
- **3-1. Cloudflare Enterprise 연동**: WAF, D1/KV 잔액 캐싱, 지오 라우팅 방화벽 가동.
- **3-2. 3대 운영 봇 가동**: 시뮬레이션 컴파일러, eBPF 자가 치유, 주단위 벤치마크 워커 활성화.
- **3-3. 볼륨 딜(Volume Deal) 협상**: 동접 1만 명 달성 시 인프라 원가 협상을 통한 마진율 극대화.
